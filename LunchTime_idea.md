# 🍚 Lunch

> **Slack 대신, 메뉴바에서 점심을 모집하다.**

**Lunch**는 회사에서 매일 반복되는 점심 모집 과정을 없애기 위해 만든 macOS 전용 생산성 앱입니다. 메뉴바(MenuBar)와 터미널(CLI) 두 가지 인터페이스를 제공하고, 두 인터페이스는 하나의 Core를 공유하며 실시간으로 점심방(Room)을 관리합니다.

---

## 왜 만드는가

회사에서 점심 한 번 먹으려면 거의 매일 같은 장면이 반복됩니다.

> "마라탕 먹을 사람?" → 관심 있는 사람이 손을 든다 → 가게가 정해져 있으면 바로 주문, 아니면 같이 고른다 → 주문 → 출발

대부분 이 과정은 Slack 채널이나 DM에서 흘러갑니다. 문제는 채팅이 이런 걸 잘 못한다는 점입니다.

- 점심 모집이 동시에 여러 개 올라오면 **뒤죽박죽 섞인다.**
- 지금 어떤 모집이 **진행 중인지 한눈에 안 보인다.**
- 누가 참여하는지 **매번 스크롤해서 세어야 한다.**
- 주문이 끝났는지 **아무도 확실히 모른다.**
- 점심 이야기가 **업무 대화 사이에 끼어든다.**

한 문장으로 정리하면 이렇습니다.

> **점심 모집이 "채팅"에 얹혀 있다.**

Lunch는 이 문제를 채팅이 아니라 **"Room 기반 상태 관리"** 로 풉니다. 점심 한 건 = Room 하나. 각 Room은 명확한 상태를 갖고, 그 상태만 보면 지금 무슨 일이 벌어지는지 즉시 알 수 있습니다.

---

## 우리가 이루려는 것

- Slack보다 **빠르게** 점심을 모은다.
- 진행 중인 점심방을 **한눈에** 본다.
- **GUI(메뉴바)와 CLI(터미널)를 모두** 지원한다.
- 하나의 Core에서 **여러 인터페이스**를 굴린다.

---

## 핵심 컨셉

한 가지만 분명히 하고 갑니다.

Lunch는 ~~"메뉴를 정해주는 앱"~~ 이 **아닙니다.**

Lunch는 **"점심 모집(Room)을 관리하는 앱"** 입니다.

메뉴를 뭘 먹을지 고민해주는 게 아니라, "지금 누가 뭘 먹으러 가는지"를 매끄럽게 굴러가게 하는 것 — 그게 이 앱의 전부입니다.

## 사용자 흐름 & 주요 기능

점심 한 건이 시작해서 끝나기까지의 여정을 따라가 봅니다.

### 🍚 1. 점심방 생성

새 점심방을 만듭니다. 필요한 건 **메뉴 하나뿐**입니다.

| 구분 | 항목 |
|------|------|
| 필수 | 메뉴 |
| 선택 | 가게 · 출발 시간 · 메모 |

```
메뉴   🍜 마라탕
가게   라화쿵부
출발   12:20
```

### 📢 2. 실시간 모집

같은 네트워크에 있는 Lunch 사용자에게 알림이 뜹니다.

```
🍜 석준님이 마라탕 점심방을 생성했습니다.
```

### 🙋 3. 참여

버튼 하나로 참여. 참여자 목록은 실시간으로 채워집니다.

```
석준 · 민수 · 지은 · 태호
```

### 🏪 4. 가게 추천

가게가 아직 안 정해졌다면, 참여자들이 후보를 올립니다.

```
라화쿵부 · 신룽푸 · 마라공방
```

### 🗳 5. 투표 (선택)

의견이 갈릴 때만. 모집장이 투표를 열어 가게나 메뉴를 정합니다. *필요 없으면 건너뜁니다.*

### ⏰ 6. 주문 마감

카운트다운을 시작하면 신규 참여가 닫힙니다. "이제 확정입니다"의 신호.

### 📦 7. 주문 완료

주문이 끝나면 참여자 모두에게 알림이 갑니다.

```
✅ 주문 완료
```

### 🚶 8. 출발

출발하면 그 점심방은 히스토리로 넘어갑니다. 오늘 점심, 끝.

---

## 여러 점심방을 동시에

Slack에서라면 뒤섞였을 여러 모집이, Lunch에서는 각자의 Room으로 나란히 존재합니다.

```
🍜 마라탕 (5명)   🍔 햄버거 (3명)   🍣 초밥 (2명)   🥩 제육 (4명)
```

사용자는 마음에 드는 방에 그냥 들어가면 됩니다.

---

## 상태(State)로 말한다

Lunch의 핵심은 "상태"입니다. 채팅을 읽지 않아도, 상태 하나만 보면 지금 어디까지 왔는지 알 수 있습니다.

| 상태 | 의미 |
|------|------|
| 🟡 모집 중 | 참여자를 모으는 중 |
| 🟢 진행 중 | 참여가 진행되는 중 |
| 🗳 투표 중 | 가게 또는 메뉴 투표 중 |
| 📦 주문 완료 | 주문이 끝남 |
| 🚶 출발 완료 | 종료(히스토리) |

---

## 두 개의 얼굴 (Interface)

Lunch는 성격이 다른 두 종류의 클라이언트를 제공합니다.

### 🖥 MenuBar App — 모두를 위한 GUI

메뉴바에 상주하며, 클릭 몇 번으로 모든 걸 처리합니다.

```
🍚 Lunch
────────────
현재 모집
  🍜 마라탕    5명 참여
  🍔 햄버거    3명 참여
────────────
➕ 점심 모집
   설정
   종료
```

### ⌨ Interactive CLI — 개발자를 위한 터미널

터미널을 떠나기 싫은 사람들을 위한, 키보드만으로 완결되는 인터페이스.

```
$ lunch
```

**메인 메뉴** — `↑↓` 이동 · `Enter` 선택 · `ESC` 뒤로

```
🍚 Lunch
──────────────────────────
❯ 현재 모집 보기
  점심 모집하기
  참여 중인 방
  히스토리
  설정
──────────────────────────
```

**현재 모집** — 진행 중인 방을 한 화면에

```
🍜 마라탕   5명   12:20
🍣 초밥     2명   12:00
🍔 햄버거   4명
```

**점심방 상세** — 참여자와 정보를 보고 바로 행동

```
🍜 마라탕
📍 라화쿵부   🕐 12:20

참여자
석준 · 민수 · 지은 · 태호
────────────
❯ 참여
  가게 추천
  투표
  뒤로
```

**생성** — 메뉴만 넣으면 끝, 나머지는 선택

```
메뉴  > 마라탕
가게  > 라화쿵부
시간  > 12:20
────────────
❯ 생성    취소
```

## 아키텍처

하나의 `LunchCore`가 모든 로직을 쥐고, 인터페이스들은 그 위에 얇게 얹힙니다. 네트워킹과 저장소도 Core를 통해 공유됩니다.

```
                    LunchCore
                        │
       ┌────────────────┼────────────────┐
       │                │                │
  MenuBar App    Interactive CLI      (Future)
       │                │
       └───────── Network ─────────┘
            Bonjour · MultipeerConnectivity
                        │
                    SwiftData
```

## 로드맵 (MVP)

작게 시작해서 단계적으로 넓혀갑니다.

| 버전 | 담는 것 |
|------|---------|
| **V1** | MenuBar App · 점심방 생성 · 참여 · 실시간 동기화 |
| **V2** | Interactive CLI · 상태 변경 · 가게 추천 |
| **V3** | 주문 마감 · 투표 · 히스토리 |
| **V4** | 즐겨찾는 식당 · 통계 · AI 메뉴 추천 · 지도 연동 · 날씨 기반 추천 |

## Slack과 무엇이 다른가

| | 기존 Slack | **Lunch** |
|---|-----------|-----------|
| 중심 | 채팅(Chat) | **Room** |
| 파악 | 상태 파악 어려움 | **상태(State) 중심** |
| 정보 | 대화가 흩어짐 | **한 방에 정리됨** |
| 접점 | 앱/브라우저 | **MenuBar · CLI** |
| 연결 | 서버 경유 | **Peer-to-Peer** |
| 동기화 | 수동으로 읽기 | **실시간 동기화** |
