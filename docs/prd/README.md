# Product Requirements

`docs/prd/`는 제품 정의 과정에서 확정된 요구사항을 구현 가능한 제품 요구사항으로 정리하는 정본 영역입니다. 현재 PRD는 2026-07-24 제품 회의의 D-29~D-40 결정을 포함합니다.

- `product-definition/`은 결정에 이른 배경과 논의 과정을 보존합니다.
- `prd/`는 제품이 제공해야 하는 기능, 범위와 수용 조건을 정의합니다.
- `policies/`는 상태 전이, 권한, 충돌, 보존과 예외 처리의 세부 규칙을 정의합니다.

PRD와 Policies가 충돌하면 제품 범위와 사용자 가치는 PRD를, 세부 동작은 Policies를 기준으로 판단합니다. 어느 쪽으로도 해소되지 않는 충돌은 임의 구현하지 않고 제품 결정을 추가합니다.

## 변경 원칙

- 구현 작업은 관련 PRD 요구사항 ID와 Policy 규칙을 연결합니다.
- PR을 올리기 전에 구현 결과가 PRD·Policies를 바꾸는지 확인합니다.
- 제품 동작이 달라졌다면 코드와 문서를 같은 PR에서 갱신합니다.
- 아직 기술 검증이 필요한 사항을 보장된 제품 동작처럼 표현하지 않습니다.
- 기술 스파이크는 확정된 사용자 결과를 바꾸지 않고 구현 방식, 수치 허용치와 실패 UX만 구체화합니다.
