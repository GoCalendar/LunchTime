# 06. 기능 후보 목록

> 상태: 제품 책임자 회의 결정을 반영한 작업 원장
>
> 입력: [01. 제품 방향](./01_product_direction.md)부터 [05. 어떻게 하면 질문](./05_how_might_we.md)까지
>
> 역할: 지금까지 나온 기능과 필수 시스템 능력을 우선순위 없이 빠짐없이 수집한다.

이 문서의 순서와 ID는 우선순위가 아닙니다. 기능이 제외돼도 ID를 재사용하지 않고 `Retired`로 남기며, 새 기능은 마지막 번호 뒤에 추가합니다.

## 상태 범례

- **방향**: 제품 책임자가 명시한 제품 방향에서 직접 나온 기능
- **기존 논의**: 초기 아이디어에서 구체적으로 다룬 기능
- **파생**: 선택한 경험을 성립시키기 위해 추가 정의가 필요한 기능
- **가설**: 유용성 또는 인터페이스 적합성을 검증해야 하는 기능
- **조사**: 기술 가능성 또는 외부 공식 지원 여부에 종속되는 기능
- **후속**: 제품 방향에는 포함하지만 MVP 이후에 다루는 기능
- **Retired**: 현재 제품 모델과 맞지 않아 제외한 기능. ID는 다시 사용하지 않음

## 관리 원칙

- 이 문서는 제품 정의 단계에서 사용하는 기능 작업 원장입니다.
- PRD로 승격된 뒤에는 채택된 기능 요구사항의 정본이 PRD로 이동합니다.
- 이후 문서는 기능을 새로 정의하지 않고 F-ID로 참조합니다.
- 상태는 우선순위가 아니며, 범위 제안은 [09. 범위 제안](./09_scope_proposal.md)에서만 다룹니다.
- 참여는 마감 전에 승인 ACK를 받아야 성공하고, 승인 이벤트가 다른 Peer에 늦게 도착해도 유효합니다. ACK가 없으면 마감 시 실패합니다. 메뉴 최신 리비전도 ACK가 필요합니다.
- 주문 완료 처리 기기는 현재 참여자 전원의 최신 승인·메뉴 ACK 리비전을 보유해야 합니다. 과거·오프라인 Peer의 미응답만으로 차단하지 않으며, 로컬 단독 예외는 생성자가 처음부터 만든 never-shared Room에만 적용합니다.
- 14:30에 사용자 쓰기를 즉시 닫고 MenuBar의 당일 Room을 제거합니다. finalization은 최대 120초이며, 잠든 Mac은 wake 시 즉시 종료를 적용하고 그때부터 최대 120초 수행합니다.
- 동기화 트리거별 세션은 최대 3회이면서 총 30초 이내이고 둘 중 먼저 도달하면 종료합니다. `동기화 중`은 활동이고 지속 상태는 `이 기기에 저장됨/복제됨/동기화됨/확인 필요`입니다.
- 참여·메뉴·마감·주문 상태 같은 운영 이벤트는 최대 14일 관리합니다. 채팅은 앱 종료 시 로컬 메모리에서 삭제하며, 현재 운영일에 살아 있는 Peer가 보유할 때만 당일 메시지를 best-effort로 복구합니다.
- 재모집용 최소 히스토리는 가게명·배민 링크·최근 주문 완료 시각·로컬 이용 횟수로 구성해 14일 이후에도 각 기기에 보존합니다. Peer에는 가게명·링크·최근 완료 시각만 공유하고 로컬 이용 횟수는 병합하지 않습니다.
- 새로운 기능이 논의되면 F-75부터 추가합니다.
