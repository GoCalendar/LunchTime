# 06. 기능 후보 목록

> 상태: 검토가 필요한 초안
>
> 입력: [01. 제품 방향](./01_product_direction.md)부터 [05. 어떻게 하면 질문](./05_how_might_we.md)까지
>
> 역할: 지금까지 나온 기능과 필수 시스템 능력을 우선순위 없이 빠짐없이 수집한다.

이 문서의 순서와 ID는 우선순위가 아닙니다. 기능이 제외돼도 ID를 재사용하지 않고 `Retired`로 남기며, 새 기능은 마지막 번호 뒤에 추가합니다.

## 상태 범례

- **방향**: 제품 책임자가 명시한 제품 방향에서 직접 나온 기능
- **기존 논의**: 초기 아이디어에서 구체적으로 다룬 기능
- **파생**: 선택한 경험을 성립시키기 위해 추가 정의가 필요한 기능
- **가설**: 유용성 또는 인터페이스 적합성을 검증해야 하는 기능
- **조사**: 기술 가능성 또는 외부 공식 지원 여부에 종속되는 기능

## 관리 원칙

- 이 문서는 제품 정의 단계에서 사용하는 기능 작업 원장입니다.
- PRD로 승격된 뒤에는 채택된 기능 요구사항의 정본이 PRD로 이동합니다.
- 이후 문서는 기능을 새로 정의하지 않고 F-ID로 참조합니다.
- 상태는 우선순위가 아니며, 범위 제안은 [09. 범위 제안](./09_scope_proposal.md)에서만 다룹니다.
- 새로운 기능이 논의되면 F-64부터 추가합니다.
