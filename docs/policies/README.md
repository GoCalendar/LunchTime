# LunchTime Policies

> 상태: 회의 결정을 반영한 정본 초안
>
> 기준 시간대: `Asia/Seoul`
>
> 역할: PRD와 구현이 따라야 하는 상태, 권한, 동기화, 보존, 보안과 화면 동작을 정의한다.

## 문서 원칙

- `docs/product-definition/`은 결정을 만들어 온 과정과 근거를 보존한다.
- 이 디렉터리의 문서는 합의된 동작의 정본이다.
- PRD는 제품이 제공할 기능을 정의하고, Policies는 같은 기능이 어떤 규칙과 예외로 동작하는지 정의한다.
- 구현 세부 기술은 바뀔 수 있지만, 이 문서의 사용자 관찰 가능 동작을 바꾸려면 Policy를 먼저 갱신한다.
- 문서에서 **필수**는 MVP가 반드시 지켜야 하는 동작, **금지**는 구현이 제공하면 안 되는 동작을 뜻한다.

## 최신 회의 결정을 적용한 해석

이 정책은 기존 제품 정의와 충돌하는 경우 다음처럼 더 최신이고 구체적인 결정을 우선했다.

| 기존 내용 | 최신 결정과 정본 해석 |
|-----------|-----------------------|
| 개인 메뉴와 옵션은 자유 형식 텍스트 | 입력값 자체는 자유로운 문자열이지만, 입력 구조는 `메뉴 이름`과 반복 가능한 `옵션`으로 고정한다. |
| 주문 완료가 Room의 즉시 종료 상태 | 주문 완료는 14:30 전까지 되돌릴 수 있는 상태 표식이다. 사용자 쓰기의 최종 경계는 14:30이며 이후 최대 120초간 종료 스냅샷을 대조한다. |
| 모집 마감 뒤에도 채팅을 계속 이용 | 방별 주문 마감 시각부터 14:30 전까지 기존 참여자가 이용한다. 14:30부터 Room과 Lounge 모두 열람 전용이다. |
| 히스토리에 완료된 방을 남김 | 재사용 히스토리에는 성공 종료한 Room의 최소 가게 정보만 장기 보존한다. 취소되지 않은 종료 Room의 참여자·메뉴·최종 상태는 모든 자동 신뢰 회사 WiFi Peer에게 최대 14일 보여주며 채팅은 포함하지 않는다. |
| 채팅·히스토리 등 데이터는 14일만 관리 | 복제용 운영 이벤트와 삭제 표식은 최대 14일 관리한다. 채팅은 현재 운영일의 프로세스 메모리에만 두고 앱 종료 시 삭제하며, 재사용 목적의 최소 히스토리는 14일 이후에도 로컬에 남을 수 있다. |
| 가게 후보와 선택적 투표 | MVP에서 한 Room은 한 가게를 나타내며 사용자는 투표하지 않고 Room에 참여한다. 참여자가 한 명뿐이어도 유효한 주문 흐름이다. |
| AirDrop, 공유 시트, 복사 또는 QR 비교 | MVP의 Mac→휴대폰 배민 링크 전달 수단은 QR 하나로 제한한다. |
| MenuBar는 검증할 인터페이스 후보 | MenuBar를 기본 진입점으로 확정하고, 상세 기능은 별도 창에서 수행한다. |
| 라운지 출시 시점 미정 | 라운지는 MVP에 포함하고 정식 명칭을 `라운지`로 확정한다. |
| 회사 WiFi와 별도의 수동 신뢰 등록 | 지원 회사 WiFi에서 발견한 LunchTime Peer를 자동 신뢰한다. Peer 목록은 가시성을 위한 UI이며 내부 악성 Peer 차단 수단은 아니다. |
| 링크 형식·도메인 검증 | 입력 문자열을 검증·정규화·경고 없이 그대로 QR에 인코딩한다. |
| 연결 실패 뒤 계속 재시도 | 동기화 세션마다 최대 3회와 총 30초를 함께 적용하고, 새 의미 있는 트리거나 수동 요청 전까지 멈춘다. |

## 정책 우선순위

서로 충돌하는 규칙은 다음 순서로 해석한다.

1. 데이터 누락으로 실제 주문에서 사람이 빠지는 일을 막는 규칙
2. 14:30 쓰기 종료, 최대 120초 최종 대조와 열람 전용 규칙
3. 사용자 본인의 참여·메뉴 의사를 보존하는 규칙
4. 편의성과 화면 단순화 규칙

중앙 서버가 없는 구조에서는 모든 상황의 완전한 전달을 수학적으로 보장할 수 없다. 따라서 연결되지 않은 기기의 정보를 추측해 `동기화됨`으로 표시하는 행위를 금지하고, 보장할 수 없는 상태를 명확하게 보여주는 것도 이 정책의 일부로 본다.
