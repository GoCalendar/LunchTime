# POL-04. macOS 화면·채팅

> 상태: 정본 초안
>
> 관련 결정: D-06, D-09, D-12~D-13, D-23~D-25, D-28, D-32, D-34~D-36, D-39, C-01, C-04, C-06, C-07

## 1. MVP Surface

MVP는 다음 macOS Surface를 제공한다.

| Surface | MVP 역할 |
|---------|----------|
| MenuBar | 오늘의 현황을 빠르게 확인하고 별도 창을 여는 기본 진입점 |
| Room Window | 참여, 메뉴, QR, 주문 상태와 Room 채팅을 처리하는 상세 창 |
| Lounge Window | 점심 주문과 무관한 대화를 나누는 별도 창 |
| History Window | 로컬 재사용 히스토리에서 새 Room 생성 흐름으로 진입 |
| Recent Room Detail | 취소되지 않은 종료 Room의 구조화된 상태를 최대 14일 동안 열람 |
| Connection / Settings | Peer 목록, 마지막 동기화와 수동 새로고침 |

- 긴 메뉴 입력, 채팅, 히스토리 편집과 연결 문제 해결을 MenuBar 팝오버 안에 구현하지 않는다.
- CLI는 MVP에서 제외한다.
- 정식 제품 용어는 `라운지`다. 휴게실, 잡담방은 UI 명칭으로 사용하지 않는다.

## 2. MenuBar

MenuBar는 항상 전체 상세 창을 대신하지 않는 요약·진입 Surface다.

### 표시할 정보

- 11:00~14:30에 오늘 생성됐고 취소되지 않은 Room
- Room별 가게 이름
- 참여자 닉네임 또는 좁은 공간에서는 참여자 수와 펼쳐보기
- 방별 주문 마감 시각
- 모집·주문 상태 아이콘
- `확인 필요` 동기화 경고

### 상태 아이콘

| 아이콘 의미 | 계산 기준 |
|-------------|-----------|
| 모집 중 | 일일 활성, 방별 마감 전, 주문 진행 중 |
| 모집 마감 | 일일 활성, 방별 마감 뒤, 주문 진행 중 |
| 주문 완료 | 일일 활성, 현재 주문 완료 표식이 유효 |
| 확인 필요 | 데이터 동기화가 불완전하거나 충돌이 있음 |

`확인 필요`는 다른 상태에 덧붙는 경고이며 사용자에게 숨기지 않는다. 사용자가 Room을 선택하면 별도 Room Window를 연다.

- 취소 Room은 즉시 MenuBar 목록에서 제거하되, 앱 창의 오늘 Room 목록에는 14:30까지 `모집 취소`·열람 전용으로 남겨 다시 열 수 있게 한다.
- 14:30에는 주문·동기화 상태와 관계없이 오늘 Room을 모두 MenuBar에서 즉시 제거한다.
- 종료·최종 확정 상태는 MenuBar에 오늘 Room을 계속 남기는 방식으로 표시하지 않는다. 취소되지 않은 Room은 Recent Room 목록에서 확인한다.

### MVP에서 제외할 MenuBar 요소

- 라운지 전용 바로가기 버튼
- 라운지 읽지 않은 메시지 배지
- Room·라운지의 정교한 읽지 않음 집계
- 팝오버 안에서의 채팅과 메뉴 편집

라운지는 MVP에 포함하지만 메인 앱 창 또는 별도 창 탐색으로 들어간다. 라운지 바로가기와 읽지 않음 표시는 후속 기능이다.

## 3. Room Window

Room Window는 다음 영역을 같은 맥락 안에 제공한다.

1. 가게 이름, 입력 문자열의 QR, 방별 마감과 Room 상태
2. 참여·참여 철회와 참여자 목록
3. 고정 템플릿 기반 개인 메뉴 등록과 참여자별 메뉴 요약
4. 주문 전 점검, 주문 완료와 되돌리기
5. Room 채팅과 주요 시스템 이벤트
6. 마지막 동기화 시각, 네 상태 중 현재 동기화 상태와 수동 새로고침

### 가게와 참여

- 한 Room에는 한 가게만 표시한다.
- 가게 후보 목록, 투표 버튼, 득표 수와 최종 가게 선정 UI를 제공하지 않는다.
- 참여자가 한 명이어도 빈 Room이나 오류 상태로 표시하지 않는다.
- 자동 신뢰 회사 WiFi Peer는 참여하기 전에도 가게, 참여자, 각자의 메뉴와 Room 채팅을 읽을 수 있다.
- 참여는 주문 의사이므로 메뉴 미등록 참여자를 주문 전 점검에서 분명하게 보여준다.
- 메뉴 작성·수정, Room 채팅 작성과 주문 상태 변경은 현재 Room 참여자에게만 허용한다.
- 주문용 메뉴 요약 복사는 주문 전 점검을 통과한 최신 ACK 리비전에만 허용한다. 누락·대기·충돌이 있으면 복사 동작을 비활성화하고 문제 대상을 보여준다.
- 같은 사용자의 두 Room 참여가 충돌하면 양쪽에 `참여 충돌`과 선택 UI를 표시하고, 선택하기 전 메뉴·주문 동작을 막는다.

### QR

- MVP에서 Mac→휴대폰 전달 버튼은 `QR 보기` 하나만 제공한다.
- 사용자가 입력한 문자열을 검증·정규화·경고 없이 그대로 QR로 로컬 생성한다.
- 링크 형식, 도메인, 리디렉션과 실제 배민 앱 이동 여부를 검증하지 않으며 안전·공식 링크 라벨을 표시하지 않는다.
- QR 생성 자체가 실패하면 QR을 표시할 수 없다는 사실만 알린다.
- LunchTime은 QR 생성·표시까지만 책임지며 스캔 이후 외부 앱 동작은 보장하지 않는다.
- AirDrop, macOS 공유 시트, 기기간 자동 전달은 MVP에서 제공하지 않는다.
- 복사는 내부 진단이나 접근성 대안으로 채택하기 전까지 모바일 전달의 공식 사용자 흐름으로 표현하지 않는다.

### 채팅과 시스템 이벤트

- 사용자 대화와 시스템 이벤트는 시각적으로 구분한다.
- 참여·철회, 가게·링크 변경, 마감 변경·도달, 메뉴 리비전 변경, 주문 완료·되돌리기, Room 취소와 동기화 경고는 시스템 이벤트로 표시한다.
- 시스템 이벤트는 전체 메뉴 내용을 채팅에 복제하지 않고 변경 주체와 종류만 알린다.
- 자동 신뢰 회사 WiFi Peer는 Room 채팅을 읽을 수 있고, 현재 참여자만 쓸 수 있다.
- 주문 완료 뒤에도 현재 참여자는 14:30 전까지 채팅을 쓸 수 있다.
- Room이 취소되거나 14:30이 되면 채팅 입력창을 비활성화한다. 취소 Room은 14:30까지, 정상 종료 Room은 현재 앱 세션이 끝날 때까지 Room Window를 열어 현재 메모리의 채팅을 읽을 수 있다.
- 메시지 ID로 중복을 제거하며 Peer에서 다시 받은 메시지를 두 번 표시하지 않는다. 동일한 메시지 집합을 보유한 Peer끼리는 가능한 범위에서 결정적 순서를 표시하되, 완전 복구나 모든 Peer의 동일한 전역 순서는 보장하지 않는다.

## 4. Lounge Window

- 라운지는 MVP에 포함한다.
- 라운지는 모든 자동 신뢰 회사 WiFi Peer에게 열려 있는 일일 대화 공간이다.
- 라운지는 Room, 가게, 참여, 메뉴와 주문 상태를 갖지 않는다.
- 11:00~14:30에만 메시지를 쓸 수 있고 14:30 이후에도 현재 앱 세션에서는 Lounge Window를 열어 현재 메모리의 메시지를 읽을 수 있다.
- 앱 종료 시 해당 기기의 라운지 채팅 캐시를 삭제한다.
- 같은 활성 일일 세션에 앱을 다시 실행하면 살아 있는 Peer로부터 당일 라운지 채팅을 받을 수 있지만 완전한 복구를 보장하지 않는다.
- 메시지 ID로 중복을 제거한다. 동일한 메시지 집합을 보유한 Peer끼리는 가능한 범위에서 결정적 순서를 계산하되, 완전 복구나 모든 Peer의 동일한 전역 순서는 보장하지 않는다.
- 라운지 채팅을 재사용 히스토리나 Recent Room Detail에 넣지 않는다.
- 읽지 않음 배지, 알림과 MenuBar 바로가기는 MVP에서 제외한다.

## 5. History Window

- 가게 이름, 최근 주문 완료 시각과 로컬 이용 횟수를 목록으로 보여준다.
- 동일 항목은 `trim(가게 이름) + 입력 링크 원문`이 모두 완전히 일치할 때만 합친다. 링크의 공백을 포함해 URL 정규화는 하지 않는다.
- 항목을 선택하면 가게 이름과 입력 링크가 채워진 새 Room 생성 화면을 연다.
- 사용자는 주문 마감 시각을 확인·입력한 뒤 Room을 생성한다. 히스토리 선택만으로 즉시 생성하지 않는다.
- History Window에서 과거 채팅, 참여자나 개인 메뉴를 보여주지 않는다.
- 사용자는 로컬 히스토리 항목을 삭제할 수 있으며, 모든 Peer에서 영구 삭제된다는 표현을 사용하지 않는다.

## 6. Recent Room Detail

- Recent Room 목록에는 현재 기기에 복구된 최대 14일 이내의 취소되지 않은 종료 Room을 표시한다.
- 자동 신뢰 회사 WiFi Peer는 해당 Room 참여 여부와 관계없이 상세를 열 수 있다.
- 상세에는 가게 이름·입력 링크, 최종 주문 상태, 참여자·개인 메뉴 스냅샷, 주요 시스템 이벤트와 최종 동기화 완전성을 표시한다.
- 최종 대조가 불완전해도 현재 확인된 상세를 숨기지 않고 `종료 · 동기화 불완전`으로 표시한다.
- 새 의미 있는 동기화 트리거나 수동 새로고침으로 유효한 늦은 이벤트를 받으면 열람 전용 스냅샷을 다시 계산할 수 있다.
- 모든 입력과 상태 변경을 비활성화한 열람 전용 Surface다.
- 채팅은 메모리 전용이므로 지속되는 Recent Room Detail에 포함하지 않는다. 같은 앱 세션에 남은 당일 채팅은 기존 Room Window에서 읽을 수 있다.
- 운영 데이터가 만료되면 해당 상세 항목을 제거한다. 재사용 히스토리의 최소 가게 레코드는 별도 정책에 따라 계속 남을 수 있다.
- 취소 Room은 Recent Room 목록과 재사용 히스토리에서 제외한다.

## 7. Connection / Settings와 복구 경험

Connection / Settings에서 인식된 Peer를 목록으로 표시한다.

| 표시 정보 | 규칙 |
|-----------|------|
| 닉네임 | 중복을 허용하고, 충돌할 때만 짧은 기기 ID를 덧붙인다. |
| 연결 상태 | `온라인` / `오프라인` |
| 마지막 확인 | 로컬 기기가 마지막으로 Peer를 확인한 시각 |
| 동기화 상태 | `이 기기에 저장됨` / `복제됨` / `동기화됨` / `확인 필요` |

- Peer 목록은 Connection / Settings에서만 제공하며 Room과 MenuBar에는 전체 목록을 넣지 않는다.
- Peer별 Room 데이터 보유·ACK 여부 또는 별도 신뢰 상태는 표시하지 않는다.
- `동기화 중`은 네 상태를 대체하지 않는 일시적인 활동 표시다.
- 현재 세션이 최대 3회 또는 총 30초 한도에 먼저 도달해 실패하면 `확인 필요`로 멈춘다.
- `새로고침`은 Connection / Settings, Room Window와 Recent Room Detail에서 실행할 수 있어야 하며 새 제한 세션을 시작한다.
- 잠자기 복귀와 네트워크 전환 때는 사용자가 누르기 전에 자동 동기화를 시작하되 결과와 마지막 확인 시각을 표시한다.
- 지원 네트워크가 아닐 때는 그 사실을 연결 상태와 구분해 안내한다.
